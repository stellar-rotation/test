# 壁画线稿提取与补全：损失函数改进报告

## 1. 项目背景与改进目标

本项目第一阶段以破损壁画 RGB 图像和损伤 Mask 作为输入，输出干净、连续、适合后续修复的灰度线稿。训练涉及 L1、Edge、SSIM、Tversky、Skeleton、Direction 和 Adversarial 七类损失。

改进前主要存在三个问题：

1. 不同损失的数值尺度和任务作用不同，动态权重没有产生明显调节。
2. 损伤区以白色背景为主，统一像素平均容易让模型通过少生成线条来降低 L1，造成 Recall 偏低。
3. 骨架和方向损失如果没有适配白底黑线数据，会受到背景、梯度正反方向和无效区域干扰。

因此，本轮改进围绕以下三个目标展开：

- 让多损失组合更符合线稿补全任务，而不是仅追求数值相近。
- 让不同图像和损伤区真实线条得到更公平、直接的监督。
- 让结构损失真正约束黑色线条的连通性和走向。

## 2. 改进总览

| 改进项 | 改进前 | 为什么要改 | 已如何改进 | 当前状态 |
|---|---|---|---|---|
| 多损失平衡 | 依赖固定权重和调节幅度很小的 DWA | DWA 系数长期接近 1，无法解决绝对尺度差异 | 重新设置基础权重，关闭 DWA，分阶段启用结构和 GAN 损失 | 已实现并完成旧方案训练 |
| 损伤区监督 | 整个 batch 统一归一化，稀疏线条主要依赖像素损失 | 背景占比大，低 L1 可能对应低 Recall；高密度图可能贡献更多梯度 | 线条感知权重、逐样本归一化、损伤区 Tversky | 已实现；后两项待完整训练验证 |
| 骨架约束 | 前景定义没有充分适配白底黑线 | 大面积白色背景可能干扰拓扑监督 | 反转为黑线前景，采用 soft-clDice | 已实现 |
| 方向约束 | 可能在背景计算，并区分相反梯度方向 | 背景没有可靠方向，同一条无向线不应因正反方向被惩罚 | 仅在 GT 边缘计算 `1-|cos|` | 已实现 |
| 训练顺序 | 多种目标较早同时参与 | 初期基础轮廓不稳定，结构损失和 GAN 容易相互干扰 | 重建、结构、GAN 分阶段加入 | 已实现 |

## 3. 改进一：多损失权重与训练顺序

### 3.1 改进前

训练同时包含多个量纲不同的损失，并使用 DWA 根据各损失下降速度生成动态系数。实际日志中，DWA 系数长期约为 `0.99～1.01`，基本没有改变各损失的相对贡献。

### 3.2 为什么需要改进

DWA 衡量的是损失下降速度，不能直接校准不同损失的绝对数值和梯度范数。即使两个损失下降速度相同，它们对共享网络参数产生的梯度也可能相差很大。

旧训练日志还显示，加权 loss 数值中 Edge 约占 `52.7%`，L1 约占 `23.8%`。这不能直接证明 Edge 梯度一定主导训练，但说明仅观察原始 loss 或 DWA 系数不足以判断真实平衡状态。

### 3.3 已如何改进

首先，根据任务作用重新设置基础权重：

| 损失 | 当前权重 | 作用 |
|---|---:|---|
| L1 | 20.0 | 保持像素忠实度和整体轮廓 |
| Edge | 8.0 | 约束边缘位置和清晰度 |
| SSIM | 4.0 | 保持局部结构 |
| Hole Tversky | 0.75 | 在损伤区平衡漏线和伪线 |
| Skeleton | 3.0 | 保持拓扑与连通性 |
| Direction | 6.0 | 保持局部线条走向 |
| Adversarial | 0.05 | 低权重改善线条自然度 |

总损失为：

```text
L_total = 20 L_l1 + 8 L_edge + 4 L_ssim
        + 0.75 L_tversky
        + 3 L_skeleton + 6 L_direction + 0.05 L_adv
```

其次，将 `DWA_ENABLED` 设置为 `False`。DWA 代码仍保留，后续只有在补充梯度范数证据后才考虑重新启用。

最后，采用分阶段训练：

| 阶段 | 启用内容 | 改进目的 |
|---|---|---|
| 初始阶段 | L1、Edge、SSIM、Tversky | 先建立稳定的像素和损伤区线条监督 |
| 约第 11 轮后 | 加入 Skeleton、Direction | 在基础轮廓稳定后优化连通性和方向 |
| 约第 31 轮后 | 加入 Adversarial | 最后进行低权重视觉细化 |

### 3.4 改进后与验证状态

固定权重和分阶段策略已经用于旧训练。日志表明模型能够稳定收敛，GAN 没有出现明显失控，DWA 关闭后也没有缺少有效动态调节的问题。

但“加权 loss 数值占比”不等于“梯度占比”。是否进一步降低 Edge 权重，仍需记录各损失在共享解码层上的梯度范数后再决定。

## 4. 改进二：损伤区线条监督

### 4.1 改进前

像素损失主要按整个 batch 的所有像素统一求和并除以总权重。损伤区内的背景与线条没有得到足够有针对性的区分，且高线条密度样本可能在一个 batch 中产生更多加权贡献。

### 4.2 为什么需要改进

验证日志提供了直接证据：

```text
Epoch 90～122 平均 Hole Precision = 0.9514
Epoch 90～122 平均 Hole Recall    = 0.7562
Epoch 90～122 平均 Hole F1        = 0.8422
```

Precision 长期明显高于 Recall，说明主要问题是漏线而不是大面积伪线。

最佳第 95 轮的 Hole L1 为 `0.1737`，高于多个后续 epoch，但 Recall 和 F1 反而最高。统计中 Hole L1 与 Recall 的相关系数约为 `0.69`。这说明模型生成更多真实线条时，像素误差可能暂时增大；单纯追求低 L1 容易偏向空白预测。

跨图像测试还表现出相反失败模式：稀疏线稿容易漏线，高密度线稿容易增粗或过提取。因此不能只对整个 batch 统一增加线条权重。

### 4.3 已如何改进

#### 4.3.1 线条感知权重

先从 GT 中提取黑色线条，并用 `5×5` 最大池化扩展到线条邻域。当前区域权重为：

| 区域 | 权重 |
|---|---:|
| 完好区域 | 1.0 |
| 损伤区背景 | 1.0 |
| 损伤区真实线条及邻域 | 4.0 |

设损伤 Mask 为 `M`，扩展后的 GT 线条区域为 `F`：

```text
W_hole = 1 + (4 - 1) * F
W = (1 - M) + M * W_hole
```

该权重图用于 L1、Edge 和 SSIM，使额外监督集中在应该恢复线条的位置，而不是整片损伤背景。

#### 4.3.2 逐样本损失归一化

改进前：

```text
L_old = sum_batch(W * L_pixel) / sum_batch(W)
```

改进后：

```text
L_i = sum_image_i(W_i * L_pixel_i) / sum_image_i(W_i)
L_new = mean_batch(L_i)
```

即每张图先独立归一化，再对 batch 求平均。这样高密度样本和稀疏样本在 batch 层面具有同等基础权重。

#### 4.3.3 损伤区 Tversky Loss

新增只在损伤 Mask 内计算的 Tversky Loss：

```text
Tversky = TP / (TP + alpha * FP + beta * FN)
L_tversky = 1 - Tversky
```

当前参数为：

```text
alpha = 0.45
beta = 0.55
lambda_tversky = 0.75
```

`beta` 略高于 `alpha`，用于适度加强漏线惩罚，同时保留对伪线的约束。该损失内部使用 FP32 累加，避免 512×512 图像在混合精度下求和溢出。

### 4.4 改进后与验证状态

逐样本归一化和 Tversky 已通过以下代码级测试：

- 逐样本归一化结果与人工计算一致。
- 完美预测的 Tversky 约为 `0.000058`。
- 全空白漏线预测的 Tversky 约为 `0.815`。
- Mask 外预测变化不影响 Tversky。
- 真实训练样本组合损失能够正常反向传播，梯度均为有限值。

这些结果只能证明实现正确，尚不能证明模型指标已经提高。必须重新训练或微调，并与旧 checkpoint 在同一固定验证集上比较 Hole Precision、Recall、F1 和视觉连通性。

## 5. 改进三：骨架与方向约束

### 5.1 改进前

线稿采用白底黑线表示。如果直接将图像灰度作为前景概率，白色背景会成为数值较大的主体。方向损失如果在整幅图上计算，也会让大面积无方向背景参与平均；此外，同一条边缘的正反梯度可能被误判为方向不一致。

### 5.2 为什么需要改进

像素、边缘和 SSIM 主要约束局部外观，不能充分表达“线条是否断裂”和“线条是否沿正确方向延伸”。但结构损失只有在前景定义和有效区域正确时才有意义，否则可能优化背景而不是黑色线条。

### 5.3 已如何改进

#### 5.3.1 Skeleton Loss

先将白底黑线反转为“线条等于 1、背景等于 0”，再通过可微形态学骨架提取计算 soft-clDice：

```text
clDice = 2 * TopologyPrecision * TopologySensitivity
          / (TopologyPrecision + TopologySensitivity)

L_skeleton = 1 - clDice
```

该损失同时考虑预测骨架是否落在目标线条内，以及目标骨架是否被预测线条覆盖。

#### 5.3.2 Direction Loss

通过 Sobel 算子计算预测与 GT 梯度，只在 GT 梯度幅值超过阈值的位置计算：

```text
L_direction = 1 - |cos(theta_pred, theta_gt)|
```

使用绝对值后，正反梯度被视为同一个无向线条方向；使用 GT 边缘 Mask 后，平坦背景不再干扰方向平均。

### 5.4 改进后与验证状态

改进后，Skeleton、Direction 和 Edge 分别负责拓扑连通性、局部方向和边缘位置，作用边界比改进前更明确。

当前模型视觉结果已经表现出较平滑、连续的线条，但尚未进行只关闭 Skeleton 或只关闭 Direction 的独立消融，因此不能把视觉提升完全归因于某一个结构损失。

## 6. 改进前后汇总

| 维度 | 改进前 | 改进后 | 预期影响 |
|---|---|---|---|
| 动态权重 | DWA 系数接近 1 | 关闭 DWA，使用任务导向固定权重 | 减少无效动态调节 |
| 损失启用 | 多目标较早共同作用 | 重建、结构、GAN 分阶段启用 | 提高训练稳定性 |
| 损伤区权重 | 背景与线条缺少针对性区分 | 仅强化 GT 线条及邻域 | 提高有效补线监督 |
| Batch 聚合 | 整个 batch 统一归一化 | 每张图独立归一化后平均 | 平衡稀疏与密集图像 |
| 前景监督 | 稀疏线条依赖像素损失 | 增加 Masked Tversky | 改善 Recall，并约束伪线 |
| 骨架前景 | 白底黑线适配不足 | 反转黑线为前景并计算 clDice | 强化连续性和拓扑 |
| 方向计算 | 背景和梯度符号可能干扰 | 仅在 GT 边缘计算无向夹角 | 约束有效线条方向 |

## 7. 建议的消融实验

为证明每项改进是否有效，建议使用相同数据划分、随机种子和初始 checkpoint，依次比较：

| 实验 | 逐样本归一化 | Tversky | Skeleton/Direction | 目的 |
|---|---|---|---|---|
| A：旧基线 | 否 | 否 | 是 | 保留当前旧 checkpoint 结果 |
| B | 是 | 否 | 是 | 单独验证逐样本归一化 |
| C | 是 | 是 | 是 | 验证 Tversky 的增量效果 |
| D | 是 | 是 | 否 | 验证结构损失的必要性 |

每组至少报告：

- Hole Precision、Recall、F1。
- 允许 2 像素偏差的 tolerant F1。
- 按线条密度分组的稀疏、中等、密集结果。
- 多随机 seed 的均值和标准差。
- 典型样本的线条连续性和伪线情况。

## 8. 汇报时可直接使用的总结

改进前，多种损失虽然同时参与训练，但 DWA 系数长期接近 1，损伤区又以背景像素为主，模型容易通过减少线条输出来降低像素误差，表现为 Precision 较高而 Recall 偏低。同时，骨架和方向损失没有充分排除白色背景及梯度方向符号的干扰。

针对这些问题，首先重新设置了任务导向的固定权重，并采用分阶段训练；其次使用 GT 线条邻域构建损伤区权重图，将 L1、Edge、SSIM 改为逐样本归一化，并加入损伤区 Tversky Loss 平衡漏线与伪线；最后将黑色线条明确设为骨架前景，并只在 GT 边缘计算无向方向误差。

目前固定权重、结构损失和训练阶段策略已经用于完整训练；逐样本归一化和 Tversky 已通过实现与反向传播测试，但其实际提升仍需通过重新训练和消融实验确认。
